Einleitung - Was ist eine JSON-API?

Zuvor haben wir bereits gelernt, dass ein Backend u.a. dafür da ist, dem Frontend Daten bereitzustellen. Das erfolgt häufig im JSON-Format.

JSON ist eine Abkürzung für JavaScript Object Notation. Also die Notation, die in JavaScript für Objekte genutzt wird. So simpel. Ein kleines Beispiel: 

{
  id: 123456,
  name: "Karl",
  age: 35,
  is_admin: false
}


JSON-APIs

Backends, die Daten für ein Frontend (oder einen anderen Backend-Service) bereitstellen, besitzen häufig eine solche JSON-Schnittstelle. Diese nennt man dann häufig JSON-API. Eine einfache Implementation eines solchen JSON-API-Backends ist Thema dieses Kapitels.

Unter einem Bestimmten Pfad, kann ein anderer Service dann diese Daten abrufen. Stellen wir uns vor, unser Backend-Service ist unter der IP 192.168.123.100 auf Port 8080 erreichbar.

Unser HTTP-Endpoint, also eine Schnittstelle, die per HTTP erreichbar ist, könnte dann beispielsweise unser obiges JSON-Objekt unter folgender URL erreichbar machen:
http://192.168.123.100:8080/api/person?id=123456
Hier wird mit dem Query-Parameter id der jeweilige Nutzer ausgewählt.

Häufig sieht man auch folgende Art des URL-Designs:
http://192.168.123.100:8080/api/person/123456



Methoden einer JSON-API

Das vorherige Beispiel zeigt eine typische Art, wie mit einem HTTP-GET-Request an die Daten angefragt werden. Andere HTTP-Methoden wie POST, PUT und DELETE.

Im Kontext einer CRUD-Applikation (Create=Erstellen, Read=Lesen, Update=Updaten, Delete=Löschen) werden die einzelnen HTTP-Methoden häufig genau dafür genutzt:

Create: POST

Senden von JSON-Daten an den Server

Read: GET

Erhalten von JSON-Daten vom Server

Update: PUT

Updaten von Daten auf dem Server

Delete: DELETE

Löschen von Daten auf dem Server
