Einfache Datentypen

Jede Programmiersprache hat eigentlich die gleichen einfachen Datentypen. Bei manchen muss man sie explizit hinschreiben (C++, Java), bei manchen werden sie implizit vom Interpreter oder Compiler detektiert (Python, JavaScript, Go). Aber tiefer als das müssen wir in diesem Kapitel gar nicht abtauchen.

Die einfachen Datentypen in Go sind Boolean, String, Integer, Float und noch ein paar andere.



Boolean: wahr oder falsch

Am ersten Datentyp sehen wir direkt die zwei Möglichkeiten, wie in Go Variablen initialisiert werden können: 

// Variante 1:
var weLoveCookies bool
weLoveCookies = true

// Variante 2:
isWeatherGood := true
isWeekend := false

Variante 1 zeigt, wie zunächst die Variable weLoveCookies deklariert wird. In Go wird der Variable dabei Standardwert des einfachen Datentyps zugewiesen. Zu den Standardwerten kommen wir am Ende des Unterkapitels.

Variante 2 zeigt, wie die Variablen in Zeile 5 und 6 deklariert und direkt mit Werten initialisiert werden.

Diese beiden Varianten gelten ebenfalls für alle weiteren einfachen Datentypen.



String: Text jeglicher Art

Text jeglicher Art wird in Go mit dem Datentyp string ausgedrückt: 

var name string
name = "Karl"


Integer: Ganze Zahlen

Will man ganze Zahlen, egal ob positiv oder negativ ausdrücken, benötigt man den Integer-Datentyp. 

var zahl int
zahl = 1000

Außer int gibt es noch diverse andere "Spezial"-Varianten wie int8, int32, int64, uint, uint8, uint32, uint64 und uintptr. Aber welcher soll den jetzt benutzt werden? Das ist zum Glück ganz einfach: Gibt es keinen besonderen Grund, eine "Spezial"-Variante zu nutzen, sollte stets einfach int verwendet werden.



Float: Gleitkomma-Zahlen

Natürlich kann man auch Gleitkommazahlen ausdrücken.

var pi float64
pi = 3.1412

Außer float64 gibt es noch float32, wobei heutzutage eigentlich stets ersterer genutzt werden sollte.



Standartwerte

In Go hat jeder einfache Datentyp einen Standardwert. Das verhindert die in manchen Programmiersprachen üblichen Probleme, dass eine Variable, auf die zugegriffen wird, obwohl sie nie initialisiert wurde. Das führt dann entweder zu Programmabstürzen oder Zugriff auf Bereiche des Arbeitsspeichers.

Die Standardwerte sind eigentlich ziemlich nachvollziehbar:

  • bool ➜ false
  • string ➜ "" (leerer String)
  • int ➜ 0
  • float64 ➜ 0.0
