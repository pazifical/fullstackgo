Kontrollstrukturen

Da das Buch Programmierkenntnisse voraussetzt, können wir dieses Thema ziemlich schnell abhandeln.

If-Else

If-Else ist ziemlich selbsterklärend und ähnlich wie in jeder weitverbreiteten Programmiersprache. Auf Klammern in der Bedingung wird verzichtet. 

randomNumber := 21

if randomNumber < 0 {
  fmt.Println("Deine Zahl ist negativ")
} else if randomNumber > 0 {
  fmt.Println("Deine Zahl ist positiv")
} else {
  fmt.Println("Deine Zahl ist 0")
}


Switch-Case

Ein ähnlich an den Haaren herbeigezogenens Beispiel gibts für Switch-Case: 

name := "Joe"

switch name {
case "Anna":
  fmt.Println("Hallo Anna, schön dich wieder zu sehen!")
case "Joe":
  fmt.Println("Ach Joe, schön dich zu sehen!")
default:
  fmt.Println("Hallo Fremder!")
}

Anders als in manch anderer modernen Programmiersprache ist das switch-case in Go jedoch ziemlich langweilig.



For-Schleifen

In Go gibt es nur eine Variante, wie man Listen durchläuft:

names := []string{
  "Karl",
  "Anna",
  "Joe",
}

for i, name := range names {
  fmt.Println(i, name)
}
// Output:
// 0 Karl
// 1 Anna
// 2 Joe

for _, name := range names {
  fmt.Println(name)
}
// Output:
// Karl
// Anna
// Joe

Das gleiche gilt auch für Key-Value-Paare:

var people = map[string]Person{
  "cool_joe_1337": Person{Name: "Joe", age: 42},
  "lizzy_twizzy": Person{Name: "Alice", age: 24} 
}

for username, person := range people {
  fmt.Println(username, person)
}

Der erste Wert des Wertepaars ist bei Listen also immer der Index, bei Key-Value-Paaren der Schlüssel.

Und ja... das ist etwas komisch. Kaum jemand, der neu mit Go anfängt, erwartet folgendes: 

names := []string{
  "Karl",
  "Anna",
  "Joe",
}
for name := range names {
  fmt.Println(name)
}
// 1
// 2
// 3

Und so ist Go nun eben mal auch keine perfekte Programmiersprache. Das ist jedoch ein Fehler, den man höchstens in den ersten Wochen macht. Spätestens, wenn man etwas mit dem Listenwert - also hier der Variable name - machen möchte, erhält man Linter- oder Kompilier-Fehler. Was ich damit meine? Folgender Code kompiliert nicht mal: 

var people = []Person{
  Person{Name: "Joe", age: 42}, Person{Name: "Alice", age: 24},
}

for person := range people {
  fmt.Println(person.Name)
}

Das liegt daran, dass die Variable person in Zeile 6 ein Integer und keine Person ist. Die Gefahr für Bugs ist in der Realität also eher klein.
